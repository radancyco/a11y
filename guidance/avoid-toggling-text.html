--- title: "Avoid Toggling Text" ---

Toggling text in a control to indicate a state change should be avoided where possible. For example, click on the following button:

Cons

  • Changing text may not announce the state change to assistive technology users.
  • Changing text removes the initial context of the control, which can be confusing for assistive technology users upon returniung to the control later, especially if aria-expanded or aria-pressed are used.

Instead of changing text, use visual affordance to convey state change. This is a well-known and quickly understood convention. For example, up and down arrows, check mark, plus and minus, etc.

Pros

  • Univeral in that it behaves the same for everyone. Disabled users hear a single state chnage, via ARIA. While non-disabled users see the single state change.
  • Using a single instruction means there is less text to translate.
  • Less code to write
  • Users are well-adept at using such interfaces. Afterall, if they knew how to open or save it, a text instruction to close or unsave it, is not really necessary.
