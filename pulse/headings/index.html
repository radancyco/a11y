Heading Reminders

  1. Headings reflect the hierarchical structure of a page's content.
  2. Assistive technology (AT) users rely on headings to understand and navigate pages. They're also beneficial for SEO.
  3. Don't choose headings based on size. Use the correct heading level for structure, and style it with CSS. Structure takes priority over aesthetics. Headings should introduce meaningful content. Use them purposefully to avoid confusion or unnecessary noise for AT users.
  4. The h1 hould describe the page's overall purpose. Content like branding statements that don't serve this purpose should be considered carefully. See Should Your Hero Banner Contain a Heading? Not Always.
  5. Use only one h1 per page.
  6. The h1 should match the page's title—verbatim is ideal, but a close match is acceptable.
  7. Nest headings logically (e.g., h1 > h2 > h3 > h3 > h2).
  8. Avoid skipping or misordering heading levels (e.g., h2 > h1 > h4 > h2 > h6).
  9. Heading that are present on the page but empty, should be removed. Hiding them via CSS is also acceptable (e.g., h2:empty { display: none; }).
  10. Don't stack headings to imply a subheading. Use the hgroup element to group a heading and subheading properly.

    Our Services

    What We Offer to Help You Succeed
