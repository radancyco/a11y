A11y Pulse

Automated Testing

Web pages must always be validated. Errors presented in WAVE and the W3C (World Wide Web Consortium) Validator are the most critical issues to address. Accessibility and QA (Quality Assurance) engineers should link directly to validation reports when logging issues.

Who is responsible? Everybody

  • WAVE Validation
  • W3C Validation
    Advanced Validation
    • - Runs validation on entire DOM (Document Object Model), including script generated content. Cannot be linked to.
    • - Not all errors may affect accessibility. Run on W3C Validator results page to remove non-WCAG issues.
  • Heading Validation
  • Review Page Images

Other Validation Tests (Not Mandatory)

  • Validate CSS
  • Validate Links
  • Validate Spelling
  • Validate Structured Data

Third-party Testing

While automated and manual testing techniques can take us far, they will never equal the lived experience of individuals who are disabled, so having people with disabilites test your websites and products is a must.

  • Access Works
  • WeCo Digital Accessibility
Screen Readers

Accessibility testing is not complete until the work has been engaged with via a screen reader. Listening to your work is just as important as visual inspection.

  • JAWS (Paid, 30 Minute Mode)
    • Keyboard Shortcuts for JAWS
  • NVDA (Free)
    • Keyboard Shortcuts for NVDA
  • VoiceOver for Mac
  • Windows Narrator
